Six steps. The first two happen automatically inside 60 seconds. You only touch step 5.

Trigger
Lead emails in

~30 sec

~5 sec

Build order

Five phases. Color shows who does what — yellow = your task, teal = my code, pink = handoff.

01
You
Provision inquiries@stewardshipcre.com + rotate Twilio token
Set up the dedicated mailbox in Workspace. Forward old johnmathewson.co lead emails as safety net. Rotate the leaked Twilio auth token.
02
Me
Add leads + lead_messages tables to Supabase
Write SQL migration. You run it once in Supabase Studio (~10 min). Schema becomes reproducible going forward.
03
Me
Build /api/leads/intake endpoint on the CRM
Receives email POSTs. Calls Claude Haiku to extract structured data, Sonnet to draft reply. Inserts row, fires auto-ack via Gmail API, fires Twilio SMS to your cell. End-to-end "10-min SLA" goes live here.
04
Me
Build /inbox UI inside CRM
List view + split detail page. Editable draft. Send / Archive / Promote-to-Deal buttons. Adds one nav tab next to Reports.
05
Both
Wire external triggers
I write Apps Script to watch the inbox + post to /api/leads/intake. You install it in Workspace and update CREXi/LoopNet/Buildout to forward to inquiries@stewardshipcre.com. Public site contact form points to the same endpoint.

Two decisions before I write code

Both small, both easy. Just want a yes/no so I don't guess.

1. Schema migrations — how do we manage them?
The CRM has no supabase/migrations/ folder today. We can keep editing schema in the Supabase Studio UI, or start a discipline of writing SQL files in the repo. The second is cheap and means six months from now you can rebuild the database from scratch in 30 seconds.
A · Keep doing it in Supabase Studio B · Adopt supabase/migrations/ folder · recommended
2. Anthropic API key — already in env?
Existing CRM routes call api.anthropic.com, so ANTHROPIC_API_KEY should already be set in Netlify. Just confirming so I don't bake a different env var name into the new endpoint.
Confirm: yes, ANTHROPIC_API_KEY is set
Once you give me the green light on those two, I write the SQL migration + the /api/leads/intake endpoint as the first deliverables. Then the /inbox UI. Then we wire the external triggers and you're live.
